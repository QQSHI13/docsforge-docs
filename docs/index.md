# DocsForge

Write your documentation in Markdown. Build a professional static site in minutes. Deploy anywhere — no dependencies, no install, no hassle.

DocsForge is a **self-contained documentation build system** that bundles everything you need into a single, portable package. Fork it, customize it, and ship documentation that looks great on every device.

## Quick start

``` bash title="Clone the starter"
git clone https://github.com/QQSHI13/docsforge-starter.git my-docs
cd my-docs
```

``` bash title="Build the site"
python docsforge build
```

``` bash title="Preview locally"
python docsforge serve
```

That's it. Your documentation site is now running at [localhost:8000](http://localhost:8000).

## Built on proven technology

DocsForge packages the power of [MkDocs](https://www.mkdocs.org) and the [Material for MkDocs](https://squidfunk.github.io/mkdocs-material/) theme into a single, dependency-free distribution. You get all the features — admonitions, code blocks with syntax highlighting, content tabs, diagrams, search, navigation, social cards — without managing Python packages or theme versions.

## Who is DocsForge for?

- **Open source projects** that want professional docs without CI complexity
- **Teams** who need documentation that works offline or in air-gapped environments
- **Developers** who want to version-control their docs toolchain alongside their code
- **Anyone** who believes dependencies should be explicit, not implicit
